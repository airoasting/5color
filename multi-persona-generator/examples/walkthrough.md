# 전체 워크플로우 예시 (이메일)

본문(SKILL.md)의 6단계 워크플로우를 처음부터 끝까지 적용한 예시. v1에서 점수가 갈리고 v2로 다듬는 과정을 그대로 보여준다. 그대로 베끼지 말고 자기 과제의 흐름을 짚는 데 쓴다.

사용자가 *"팀 리더한테 다음 주 정례 30분 미루는 메일 써줘"*라고 한 상황이다.

## 캐스팅

```
BLACK: 비즈니스 라이터. 15년차 이상. 한국어 내부 메일. 정중하면서 효율적인 톤. 본론은 1문단에. "송구합니다"는 한 번만 쓴다.

RED [의도 전달]: 일잘러 PM 출신, 사내 커뮤니케이션 컨설턴트. 액션이 1문단 첫 문장에 박혀있는지를 본다. 본다: 요청과 액션의 1문단 위치, 마감 명시. (9점 = 받는 사람이 미리보기만 보고 30초 안에 회신 결정.)

SILVER [톤 적합성]: 사내 정치를 길게 본 시니어 매니저. 거리감의 미세 차이를 본다. 본다: 사과의 적정량, 친밀함의 과부족. (9점 = 받는 사람이 *"매너 있네"* 한 번 떠올린다.)

BLUE [분량, 간결성]: 카피라이터 출신 에디터. 한 문장도 아까워한다. 본다: 사족 문단, 중복 정보. (9점 = 한 단어도 더 뺄 게 없다.)

GOLD: 팀 리더. 월요일 오전 9시 5분, 출근 직후 메일함을 위에서 아래로 훑는다. 손에 텀블러. (9점 = 미리보기만 보고 *"답하면 끝"*이라고 판단.)
```

```

OOO 리더님께,

해당 일자 오후 3시에 외부 파트너사와의 미팅이 새롭게 잡히게 되어, 정례 시간과 일정이 겹치게 되었습니다. 송구합니다.

가능하시다면 정례 시간을 오후 4시 또는 4시 30분으로 변경할 수 있을지 검토 부탁드립니다. 다른 팀원분들 일정도 제가 다시 정리해서 공유드리도록 하겠습니다.

확인 부탁드립니다.
감사합니다.
[이름]
```
