## 남은 것 셋. 판단이 필요하다

### 1. 규격과 카드가 다른 트리거 규칙을 쓴다

| 자리 | 값 |
|---|---|
| SKILL.md 출력 템플릿 | 합격선에서 **0.5**점 이상 낮으면 v2 질의 |
| `references/examples.md` (SKILL.md가 지정한 출력 검증용 ground truth) | **0.5** |
| 카드 26장 | 합격선 **− 0.3** 미만이면 v2 질의 |
| 0.5를 쓰는 카드 | **0장** |

규격과 정본이 한쪽을 말하고 코퍼스 전체가 다른 쪽을 한다. 앞으로 스킬이 새 카드를 만들면 기존 26장과 어긋난 규칙이 나온다.

**어느 쪽이 맞는지는 사각지대가 가른다.** 합격선 9.5인데 점수가 9.3이면 어떻게 하는가.

- 0.5 규칙. 9.0 이상이라 v2 질의를 안 하고, 9.5 미만이라 합격도 아니다. **무엇을 하라는지 규격에 없다.**
- 0.3 규칙. 9.2 이상이라 v2 질의를 안 한다. 같은 사각지대가 더 좁게 남는다.

카드 두 장이 이 구멍을 각자 다르게 메웠다.

- p27 위기 대응 메시지. "9.4 이상 9.7 미만은 한 자리만 더 손보면 합격이므로 그 자리를 짚어 **같은 라운드에서 즉시 보정**한다."
- p29 보도자료. "합격선 9.5 미만은 모두 미합격이며 **별도의 사각지대 구간을 두지 않는다.**"

나머지 24장은 사각지대를 정의하지 않는다.

**제안.** 규격을 0.3으로 옮기고 사각지대의 행동을 못 박는다. p27의 해법이 낫다고 본다. 합격선에 0.2 못 미치는 산출물을 사용자에게 물어보는 것은 왕복 비용만 늘린다. 세 구간으로 정리하면 이렇다.

```
합격선 이상            -> 합격
합격선 - 0.3 ~ 합격선   -> 그 자리를 짚어 같은 라운드에서 즉시 보정. 묻지 않는다
합격선 - 0.3 미만       -> "v2를 만들까요?" 사용자에게 묻는다
```

고칠 자리는 SKILL.md 출력 템플릿 4번·5번, `references/examples.md` 5곳, 사각지대를 안 적은 카드 24장이다.

**증상은 둘이다.**

- 사용자가 갤러리에서 본 이름("재무 모델링")을 스킬에 치면 매핑 행을 이름으로 못 찾는다. fallback 5단계로 합성 캐스팅이 나온다. 갤러리에는 잘 만든 카드가 있는데 즉석 캐스팅은 그것을 못 쓴다.
- SKILL.md가 "매핑 표의 합격선 열이 정본"이라 해놓고 그 행을 이름으로 못 찾는다. **이번 세션에 내가 직접 겪었다.** 합격선을 정렬하면서 "경영 이슈 진단 → 컨설팅 덱"을 커밋 이력을 뒤져서 알아냈다. 다음 사람도 같은 값을 치른다.

**제안.** references 매핑 행에 새 이름을 별칭으로 붙인다. `| 엑셀 모델·재무 모델링 |`처럼. 옛 이름도 사용자가 칠 수 있는 말이라 지우지 않는다. 둘 다 같은 행으로 간다.
