# 5-Color Harness

[![5-Color Harness](docs/assets/thumbnail.png)](https://5color.vercel.app)

쓰시는 분이 문서 종류를 한 단어만 말하면, 그 문서를 잘 쓰게 만드는 Claude·ChatGPT 프로젝트 지침 한 벌을 즉시 만들어 드립니다. 이메일, 보고서, PPT, 계약서, 이력서 어느 것이든 됩니다.

이 지침을 프로젝트에 한 번 붙여 두면, 그 뒤로 그 프로젝트가 내놓는 모든 문서가 다섯 전문가의 평가를 자동으로 거칩니다. 초안을 쓰는 실무자 한 명과, 논리·전문성·공감·독자 시선으로 채점하는 비평가 네 명입니다.

**라이브 데모** · [5color.vercel.app](https://5color.vercel.app)

즉석 캐스팅 외에, 경영 리더가 자주 쓰는 35개 영역을 미리 짜 둔 카드 갤러리도 함께 있습니다. 카드 한 장이 곧 그 영역용 프로젝트 지침 한 벌입니다. 아래 [카드 갤러리](#카드-갤러리)에서 확인하실 수 있습니다.

---

## 5색 메타포

| 색 | 역할 | 정체 |
|----|------|------|
| BLACK | 수행자 | 그 작업 영역 베테랑 15년차 이상 |
| RED | 이성 비평가 | 논리, 근거, 결론의 타당성 |
| SILVER | 분야 전문가 비평가 | 베테랑이 첫눈에 짚는 디테일 |
| BLUE | 공감 비평가 | 수신자의 감정, 톤, 진정성 |
| GOLD | 독자 | 시간·장소·행위·심리로 정의된 한 사람 |

세 비평가는 사고 모드가 서로 다릅니다. 한 색이 다른 색의 자리를 침범하면 그 발언은 무효입니다. GOLD는 인구통계가 아니라 상황으로 정의합니다. 예를 들어 "월요일 9시 30분, 이사회 직전에 5장짜리 보고서를 처음 펼친 CEO"입니다.

자세한 캐스팅 9원칙과 출력 템플릿은 [SKILL.md](SKILL.md)에 있습니다.

## 60초 시작

코드도 설치도 필요 없습니다. 쓰시는 Claude나 ChatGPT에 지침 한 벌을 붙여 두면 끝입니다.

1. 카드 갤러리 [5color.vercel.app](https://5color.vercel.app)를 엽니다.
2. 만들려는 문서 종류(이메일, 보고서, 계약서, 이력서 등)의 카드를 고릅니다.
3. 카드에 담긴 지침을 복사해, 쓰시는 Claude나 ChatGPT의 "프로젝트" 지침 칸에 붙이고 저장합니다.
4. 그 뒤로 그 프로젝트에 무엇을 맡기든, BLACK이 초안을 만들고 RED·SILVER·BLUE·GOLD 네 전문가가 10점 만점으로 함께 평가합니다.

갤러리에 없는 문서 종류가 필요하면, 아래 개발자용 방식으로 한 단어만 쳐도 그 자리에서 전용 지침을 만들어 냅니다.

### Claude Code로 쓰기 (개발자용)

터미널에서 스킬을 한 번 설치합니다.

```bash
git clone https://github.com/airoasting/5color.git ~/.claude/skills/5color
```

세션에서 문서 종류를 한 단어로 칩니다. 예를 들어 "이메일"입니다. 그 영역 전용 지침이 한 벌 나오는데, 분량은 한국어로 3,000자에서 5,500자, 영문으로 1,400에서 2,600단어입니다. 그대로 복사해 Claude나 ChatGPT 프로젝트의 지침 칸에 붙이면 위 4번과 똑같이 작동합니다. claude.ai 웹에서 쓰실 때는 [SKILL.md](SKILL.md)의 본문을 지침 칸에 붙여도 같은 결과가 됩니다.

## 트리거

작업 영역을 한 마디라도 말하면 즉시 캐스팅합니다. 되묻지 않습니다.

- 한국어 단어 하나: "이메일", "보고서", "PPT", "계약서", "이력서"
- 영문 단어 하나: "email", "deck", "memo", "resume"
- 문장: "외부 비즈니스 이메일 지침", "make a system prompt for cover letter"

영문 트리거는 [`references/english-augment.md`](references/english-augment.md)를 자동으로 함께 읽어, BLACK과 GOLD를 영어 환경 인물로 캐스팅하고 평가도 영문으로 진행합니다.

복합 산출물, 예를 들어 "영업 덱과 발표 스크립트"는 본체 한 영역을 정하고 나머지를 BLACK의 제약으로 흡수합니다. 시리즈물이면 캐스팅에 고정 자산 한 줄이 붙습니다.

라우팅이 갈릴 수 있는 입력은 분기 규칙을 따릅니다. 랜딩 페이지는 마케팅의 웹사이트 제작 행으로, 이미지 프롬프트는 이미지 제작 행으로 흡수합니다. 영업 덱과 피치덱은 외부 커뮤니케이션으로, 영업 제안서와 RFP는 의사결정·전략으로 갈립니다.

다음은 트리거하지 않습니다. 이미 만든 산출물을 평가·요약·번역·계수하는 일, 단순 사실 검색, 기계적 변환입니다. 이 스킬은 system instruction을 만드는 도구이지, 산출물 자체를 평가하는 도구가 아닙니다.

## 카테고리·합격선·종결 체

상세한 표는 [SKILL.md](SKILL.md)에 있습니다. 요약은 이렇습니다.

- **카테고리 8개.** 의사결정·전략, 분석·보고서, 외부 커뮤니케이션, 마케팅, 내부 커뮤니케이션, 법무 검토·규제 대응, 커리어·상담, 소셜미디어·글쓰기입니다. 표에 없는 과제는 외부 커뮤니케이션을 최종 fallback으로 씁니다. 이 여덟은 스킬이 캐스팅할 때 쓰는 축이라 `references/` 파일 구조와 같습니다. 아래 카드 갤러리의 일곱 카테고리는 리더가 어느 방에 들어가는지로 가른 별개의 축입니다.
- **합격선 3군.** 9.0은 시·소설·블로그처럼 작가의 톤이 자산인 영역이고, 9.5는 표준이며, 9.7은 계약서·이사회·IR·위기 대응처럼 결함이 곧 손실인 영역입니다.
- **종결 체.** 작업 영역마다 다릅니다. 한 산출물 안에 두 체가 섞이면 위반입니다.
- **em dash 0개.** 산문 본문에 em dash가 한 개라도 있으면 자동 실패입니다. 수 사이의 범위 표기는 예외로, 한국어는 물결표를, 영문은 하이픈이나 "to"를 씁니다.

## 카드 갤러리

스킬이 작업 영역을 즉석에서 캐스팅한다면, 카드 갤러리는 경영 리더가 자주 쓰는 35개 영역을 미리 짜 둔 라이브러리입니다. 카드 한 장이 곧 그 영역용 프로젝트 지침 한 벌입니다. 카드를 열어 5인 페르소나와 합격선을 확인하고, 지침을 복사해 프로젝트 박스에 붙이면 됩니다. 갤러리는 [5color.vercel.app](https://5color.vercel.app)에서 바로 열 수 있습니다.

카테고리는 대상과 목적 하나의 축으로 갈랐습니다. 리더가 그 문서를 들고 어느 방에 들어가는지가 기준입니다.

| 카테고리 | 카드 수 | 리더가 서는 자리 |
|---|---|---|
| 전략·의사결정 | 5 | 판단 근거를 모으고 방향을 정합니다 |
| 이사회·투자자 대응 | 5 | 이사회와 주주 앞에 섭니다 |
| 파트너십·협상 | 5 | 딜 테이블에 앉고 사인 여부를 정합니다 |
| 세일즈·마케팅 | 5 | 고객에게 팔고 시장에 알립니다 |
| 조직·리더십 | 6 | 사람을 이끌고 어려운 말을 전합니다 |
| 위기 대응·PR | 3 | 사고 두 시간 안에 회사 입장을 잡습니다 |
| 퍼스널 브랜딩 | 6 | 리더 개인의 이름으로 글을 냅니다 |
| **합계** | **35** | |

카드 일곱 장에는 별 배지가 붙습니다. 리더가 남에게 못 맡기고 직접 붙잡아야 하는 자리, 그리고 틀리면 손해가 큰 자리부터 골랐습니다.

이사회 보고서, 리서치·자료조사, 협상 전략 브리프, 계약서 검토, 타운홀·전사 메시지, 위기 대응 메시지, AI 글 자연스럽게 다듬기 일곱 장입니다. 선정 기준과 점수는 [`output/5color_pick_eval_20260715.md`](output/5color_pick_eval_20260715.md)에 적어 두었습니다. 목록은 손으로 고르지 않습니다. `scripts/select_picks.py`가 점수표에서 유도합니다.

35장 전원이 기계 검사 9종을 통과합니다. 코드펜스와 산문 em dash, 브래킷 잔존, 분량 밴드, 끝줄 안내, 비평가 톤 중복, 자평 문구, 내부 어휘 유출, 다른 산출물로 넘김을 봅니다. [`scripts/validate_output.py`](scripts/validate_output.py)로 직접 돌려 보실 수 있습니다. 카드 채점과 개선 리포트는 `output/` 폴더에 있습니다.

## 폴더 구조

저장소 루트가 곧 스킬입니다. `SKILL.md`와 `references/`·`evals/`가 스킬 본체이고, 카드 갤러리는 `docs/`에, 이 README는 루트에 있습니다.

```
5color/  (저장소 루트 = 스킬)
├── SKILL.md                  스킬 본문
├── README.md                 이 파일
├── LICENSE                   Apache License 2.0
├── references/               카테고리별 매핑·문체·금지 뱅크
│   ├── decision-strategy.md  의사결정·전략
│   ├── analysis-report.md    분석·보고서
│   ├── external-comm.md      외부 커뮤니케이션
│   ├── marketing.md          마케팅
│   ├── internal-comm.md      내부 커뮤니케이션
│   ├── legal-regulatory.md   법무 검토·규제 대응
│   ├── career-counseling.md  커리어·상담
│   ├── social-writing.md     소셜미디어·글쓰기
│   ├── english-augment.md    영문 트리거 시 자동 추가
│   └── examples.md           풀 출력 예시 5종
├── evals/                    트리거 정확도·출력 품질 평가 셋 (6케이스 × 3회)
├── scripts/                  기계 검사와 카드 도구
│   ├── validate_output.py    출력 직전 기계 검사 9종
│   ├── select_picks.py       점수표에서 별 배지 카드를 유도
│   └── score_cards.py        카드 35장 채점
├── docs/                     카드 갤러리 (라이브: 5color.vercel.app)
│   ├── index.html            갤러리 페이지
│   └── assets/               이미지·폰트·로고
└── output/                   카드 채점·개선 리포트
```

작업 영역을 추가하려면 해당 카테고리의 `references/` 파일만 손보면 됩니다. 처음 쓰실 때는 [`references/examples.md`](references/examples.md)의 풀 예시 5종으로 톤과 구조를 익히실 수 있습니다.

## 트러블슈팅

- **5색 평가가 자동으로 돌지 않습니다.** 마크다운에 `## 워크플로우` 섹션이 빠짐없이 들어갔는지 확인합니다.
- **결함이 곧 손실인 영역인데 금지가 헐겁습니다.** "이 영역은 합격선 9.7이니 직접 인용 금지를 네다섯 개로"라고 한 줄 던지면 다시 생성합니다.
- **영문 트리거인데 한국어가 나옵니다.** "in English"를 명시하거나, [`references/english-augment.md`](references/english-augment.md)가 있는지 확인합니다.
- **페르소나가 서로의 자리를 침범합니다.** "RED는 이성, BLUE는 공감만"이라고 한 줄 던지면 다시 캐스팅합니다.
- **em dash가 보입니다.** "본문 em dash 0개로 다시"라고 한 줄 던집니다.

평가 셋과 60초 점검법은 [`evals/EVALS_GUIDE.md`](evals/EVALS_GUIDE.md)에 있습니다.
